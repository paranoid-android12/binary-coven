SECTION 1
GETTING STARTED.
You need just two things to begin: an account and a session code from your instructor. Think of the code as your ticket into the classroom.
1.1 Create your account
1
Enter the session code. A green check means you’re good; a red X means it’s expired or wrong.
2
Fill in your email, username, and password. The password needs one uppercase letter, one number, and one special character.
3
Click Submit. A brand-new username jumps straight to email verification.
4
Check your inbox for a 6-digit code and type it in. No email? Wait 60s and hit Resend.
5
Verified! Your account is created, your game state loads, and you’re in.
THE STUDENT PORTAL LOGIN
TIP: ONE ACCOUNT, EVERY SESSION

Got a new session code next week? Just log in with the same username and password, and Binary Coven links your account to the new session automatically.

JOIN, RETURN & RECOVER
1.2 Join a session

Sessions open and close on a schedule and may have a student cap. If your code is expired or the room is full, the app tells you. Just ask your instructor for a fresh one.

1.3 Log in again

Returning player? Enter your username, password, and the current session code. No email verification needed the second time around.

1.4 Forgot password

Enter your username and session code. A reset code is emailed to you. Type it in, then choose a new password.

i Reset your password in three steps
1
On the login screen, enter your username and session code.
2
A reset code is sent to your registered email (shown partly masked for privacy).
3
Enter the code and set a fresh password and get back to coding.
NOTE

Session codes have time windows: scheduled, active, or expired, and sometimes a maximum number of students. Only an active code with room to spare will let you in.

SECTION 3
THE QUEST SYSTEM.
Quests are how you learn. Each one teaches a concept (sequencing, conditionals, loops, functions, optimization) by making you write code that moves Qubit through the world.
THE QUEST LOG: THREE TABS
3.1 Three tabs
AVAILABLEQuests you can start right now
ACTIVEYour current quest, one at a time
COMPLETEDFinished quests, fully replayable
Every card shows difficulty, category, time estimate, the concepts it teaches, and any prerequisites:
BEGINNER INTERMEDIATE ADVANCED
3.2 Phases & objectives

Quests split into phases, and each phase has objectives. Clear the objectives and the phase advances with a bit of dialogue. Clear the final phase and the quest (plus anything it unlocks) is done. Wrong answer? The world resets to the phase start so you can try again. No penalty. Experiment freely.

3.3 Cancel & restart

Cancel an active quest any time to try a different one, or restart a finished quest to practice again. All progress is saved to the server and persists across logins.

NOTE: LOCKED QUESTS

If your instructor enables prerequisites, some quests stay locked until you finish the ones before them. That’s by design: the concepts build on each other.

SECTION 4
THE PROGRAMMING TERMINAL.
This is the heart of Binary Coven: a Monaco code editor (the same engine behind VS Code) with syntax highlighting and autocomplete. Here’s where you command Qubit.
THE CODE EDITOR: PROGRAM TAB
4.1 Open the editor

Click Qubit, then the Program tab. The sidebar lists your functions; the editor shows the selected one. Execution always starts at main().

4.2 Write & run

Binary Coven uses Python-like syntax. Never coded before? The quests teach you step by step. Press Run and watch Qubit move in real time. Mistakes appear in the Error Display with line numbers.

4.4 Create your own functions
1
Click the + button in the sidebar.
2
Give it a unique name (duplicates are rejected).
3
A new tab opens with a stub: def name(): pass
4
Write the body, then call it from main(), just like Python.
Delete any function with the trash icon. The main() function stays. It is your entry point.

4.6 Energy management

Every action drains the battery: moving, mining, planting, harvesting. At zero energy, Qubit stops. To refill, move onto a dynamo tile and run crank(). Planning efficient routes is itself a programming lesson.

4.7 Drones

Later you’ll meet drones, separate programmable entities with their own code windows. Program a drone and you’re editing its functions, not Qubit’s. Drones always run their own main().

A DYNAMO: STAND HERE & CRANK()
TIP: THINK LIKE AN ENGINEER

Watch the energy bar before you commit to a long route. A short detour to a dynamo beats stalling halfway across the map.

SECTION 5
SAVING YOUR PROGRESS.
Binary Coven saves in two tiers: one fast, one permanent. Knowing the difference keeps your hard work safe.
5.1 The two-tier save
BROWSER While you play, your state lives in your browser for speed.
SAVE Click Save, everything syncs to the server database.
LOAD Click Load, your latest server save is restored.

Your quest progress (every phase and objective) also saves to the server. Close the tab, return tomorrow, and pick up exactly where you left off.

THE GAME MENU: SAVE & LOAD
TIP: SAVE OFTEN

Clearing your browser cache wipes the fast local copy. The server save is permanent, so click Save regularly, especially before you log out.

SECTION 7
MANAGING SESSIONS.
7.1 Create a session

Go to /admin/sessions → Create Session:

1
Session Code: type your own or leave blank to auto-generate.
2
Duration: validity window in hours or days. Default: 7 days.
3
Max Students: optional cap. Blank means unlimited.
4
Quest Selection: filter, Select All, or pick individually.
5
Enforce Prerequisites: on for ordered unlocking, off for free choice.
THE CREATE-SESSION DIALOG
7.2 Session lifecycle
STATUSWHAT IT MEANS
SCHEDULEDCreated but not yet open, students cannot join
ACTIVEInside its window, students can join and play
EXPIREDPast its window, students can no longer join
INACTIVEManually switched off by an admin
Search sessions by code, filter by status, and deactivate them from the list view.
TIP: BUILD A LESSON PLAN

For a structured class, enable prerequisites and select only this week’s quests. Students then progress through them in the intended order.
